블로그 글을 마크다운으로 쓰고 `git push` 한 번이면 Blogger에 자동으로 올라가는 시스템을 만들었다. **지금 읽고 있는 이 글도 그렇게 발행됐다.**

## 어떻게 동작하나

1. `posts/` 폴더에 마크다운 파일을 쓴다 (제목·태그·공개여부는 상단 frontmatter)
2. GitHub에 push 한다
3. GitHub Actions가 Blogger API v3로 자동 발행한다

중복 발행을 막기 위해 각 글의 상태(post ID, 내용 해시)를 파일에 기록해 둔다. 그래서 **새 글은 생성**하고, 이미 올린 글을 고치면 **업데이트만** 한다. 같은 글이 두 번 올라가는 일이 없다.

## 기술 스택

- **Node.js** — 의존성을 최소화하려고 googleapis SDK 없이 `fetch`로 REST를 직접 호출
- **markdown-it** — 마크다운 → HTML 변환
- **GitHub Actions** — push를 감지해 발행 실행

## 삽질 포인트 3가지

**1. Blogger는 API 키가 아니라 OAuth가 필요하다.**
남의 계정(=내 블로그)에 글을 쓰는 행위라서, "이 사람이 허락했다"는 증거가 필요하다. 그래서 리프레시 토큰을 한 번 발급받아 CI에서 재사용한다.

**2. OAuth 동의 화면이 "테스트" 상태면 토큰이 7일 만에 만료된다.**
자동화가 어느 날 조용히 멈추는 대표적인 함정. 앱을 "프로덕션"으로 게시하면 해결된다.

**3. 토큰은 블로그 소유 계정으로 발급해야 한다.**
다른 계정으로 받으면 읽기는 되지만 발행에서 `403 Forbidden`이 뜬다.

## 결과

이제 글 쓰는 흐름은 이렇게 단순해졌다.

```
posts/ 에 .md 작성  →  git push  →  끝
```

터미널에서 바로 올리고 싶으면 `npm run publish` 한 줄이면 된다.
